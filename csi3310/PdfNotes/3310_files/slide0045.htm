Simple Batch Systems
nWere the first operating systems (mid-50s)
nThe user submits a job (typically, cards) to a computer operator
nThe computer operator places a batch of jobs on a input device (e.g. card reader)
nA special program, the monitor, manages the execution of each program in the batch
nResident monitor is in main memory and always available for execution
nMonitor utilities are loaded when needed
nOnly one program at the time in memory, which is executed to the end before the next one.
