Process Suspension
nExplicit load control requires that we sometimes swap out (suspend) processes
nPossible victim selection criteria:
uFaulting process
Fthis process may not have its working set in main memory so it will be blocked anyway
uLast process activated
Fthis process is least likely to have its working set resident
uProcess with smallest resident set
Fthis process requires the least future effort to reload
uLargest process
Fwill yield the most free frames
