Execution within User Processes
nOS operates mostly as it was a regular user, in fact OS function appear as functions of the users that invoke them: shared address space
nVirtually all OS code gets executed within the context of a user process
nOn Interrupts, Traps, System calls: CPU switches to kernel mode to execute OS routine within the context of user process (mode switch) nControl passes to process switching functions (outside processes) only when needed (eg: interprocess synchronization, process scheduling…)
