Interrupt Handler
nIs a program that determines nature of the interrupt and performs whatever actions are needed nUpon interrupt, control is transferred to this program nControl must be transferred back to the interrupted program so that it can be resumed from the point of interruption nThe point of interruption can be anywhere in the program (except where interrupt inhibited). nThus: must save the state of the program (content of PC + PSW + registers + ...)
