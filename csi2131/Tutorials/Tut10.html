CSI 2131 Tutorial #10

Here is some stuff to help you prepare for the exam. Try to do the questions without looking at the answers.

Disclaimer: the stuff in this review covers only the second half of the course. See also tutorial #4 to review material from the first half of the course. The final exam covers all of the material from the whole semester.

The exam is closed-book, no notes, no cheat sheets, no calculators, no walkie-talkies, no computers, no copying, no cheating, no taking a dive.

The exam counts mark-for-mark toward the final grade (50 marks on the exam translate to 50 marks toward the final grade).

There are several questions on the exam, worth different numbers of marks. You should plan your time according to the number of marks allotted to the question (for example, don't spend more than 35 minutes on a question worth 10 marks).

Some questions ask you to create (an algorithm, for example), while some questions just ask you to do (apply a technique, for example).


  1. What is a fixed length code? What is a variable length code? Give an example of each.


  2. Give a Run Length Encoding of the string: "ABBCCCDDDDEEEEEF.........GHI".


  3. Give a Huffman Code of the string: "six_stick_shifts_stuck_shut".


  4. Encode the following string using Adaptive Huffman encoding: "yabbadabbadoo"


  5. Encode the following string using LZW encoding. Assume that the dictionary already contains the ASCII codes 0-255 in the first 256 entries: "nice_mice_on_ice"


  6. Decode the following sequence using LZW decoding. Assume that the dictionary already contains the ASCII codes 0-255 in the first 256 entries: <116><104><97><100><95><257><259><97><95><100><258>


  7. What are the steps of any general cosequential processing algorithm?


  8. What is storage compaction?


  9. What is dynamic reclamation?


  10. What are First Fit, Best Fit and Worst Fit?


  11. Build a heap from the following records:
    30 24 16 28 23 20 7 21 14 5 30 22 15


  12. What is a selection tree?


  13. What is multi-step merging?


  14. Do a heapsort with replacement selection for the following records if we only have room for 4 records in the heap:
    30 24 16 28 23 20 7 21 14 5 30 22 15


  15. What is an index file?


  16. What is a secondary index?


  17. What is an inverted list?


  18. Add the key M to the following B-tree.


  19. Delete the key S from the following B-tree.


  20. Delete the key U from the following B-tree.


  21. Add the key Y to the following B* tree.


  22. What is hashing?


  23. What is a hashing function?


  24. What is a collision?


  25. Name two desirable properties of a hashing function.


  26. What is a bucket?

Now here are even more things not mentioned in the review that you may also want to bone up on:

  • merge sort, k-way merge
  • paged binary trees
  • creating hashing functions
  • collision avoidance
  • collision resolution algorithms

  • extendible hashing
