  1. What is a fixed length code? What is a variable length code? Give an example of each.

    A fixed length code is a map from one set of symbols (A) to another set of symbols (B) such that all symbols in B are represented using the same number of bits.

    A variable length code is a map from A to B in which symbols in B are represented with different numbers of bits.

    Examples of fixed length codes are RLE and LZW. Examples of variable length codes are Morse and Huffman.

  7. What are the steps of any general cosequential processing algorithm?

    For k files...

         open all the files F1..Fk
     read a record ri from each file Fi
     while there are records left in any file
        find the record rmin with the minimum key value
        do something to rmin
        read the next record from file Fmin
     endwhile


  8. What is storage compaction?

    In a file with records marked as deleted, read the file sequentially writing out all records except those marked as deleted.


  9. What is dynamic reclamation?

    A data structure (such as a stack) is kept within the data file linking all the deleted records. Records marked as deleted can be reused for newly added records by traversing the data structure, eliminating the need to search for deleted records.


  10. What are First Fit, Best Fit and Worst Fit?

    A file has variable length records and some records marked as deleted. We want to reuse the records marked as deleted to store newly added records. The record to be added may be too big to fit in the space left by some of the deleted records. We must find a deleted record big enough to hold the new record.

    First Fit: use the first deleted record that's big enough to hold the new record (either the first in the file, or the first in the data structure, if dynamic reclamation is being used).

    Best Fit: use the smallest deleted record that's big enough to hold the new record.

    Worst Fit: use the biggest deleted record that's big enough to hold the new record.

  12. What is a selection tree?

    In the K-way merge we saw that finding the minimum key among K records within the merge algorithm could be very expensive (O(K)). A selection tree is a binary tree where the leaves are keys from each of the K records. The parent of each pair of keys is the smaller of the two keys. The selection tree allows us to find the minimum of K keys in O(log2K) each time through the merge loop.


  13. What is multi-step merging?

    When merging a very large number of files, it may be advantageous to merge the files in smaller batches, then merge the merged files.

  15. What is an index file?

    An index file is a file created from a data file. Records in the index file consist of a key and a reference field. The reference field indicates the location in the data file of records having the given key.


  16. What is a secondary index?

    A secondary index is an index file in which the index records consist of secondary key values as the key and primary key values as the reference field. Looking up a secondary key value in the secondary index gives the primary keys of records having that secondary key value.


  17. What is an inverted list?

    Since secondary keys are not necessarily unique, there may be several records (with different primary keys) for each secondary key value. An inverted list is a secondary index in which the key is a secondary key value and the reference field is a list of all the primary key values of records having that secondary key value.

  22. What is hashing?

    Hashing is storing records in predictable locations in a data file to allow direct access.


  23. What is a hashing function?

    A hashing function is a formula that maps primary key values onto positions in the hash table.


  24. What is a collision?

    A collision occurs when a hashing function maps two different key values onto the same hash value.


  25. Name two desirable properties of a hashing function.

    1. A hashing function should produce an even distribution of values.
    2. The range of a hashing function should cover the whole range of the hash table.


  26. What is a bucket?

    A bucket is a hash table entry capable of holding more than one record.
